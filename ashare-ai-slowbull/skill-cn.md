# A股AI硬件上游二线慢牛标的筛选

说明：这是当前 skill 的中文阅读版。Codex 实际自动触发仍以 `SKILL.md` 为准；本文件用于人工查看、维护和同步中文规则。

## 核心原则

本 skill 用于构建研究观察池，不用于给出直接买入或卖出建议。始终坚持以下核心逻辑：

```text
不追AI硬件头部加速票；
不把服务器整机当作A档核心；
从今日成交金额前200中，
寻找AI硬件上游方向里成交活跃、趋势慢牛、基本面改善、涨幅尚未透支的二线补涨标的。
```

真实筛选时必须使用当前市场数据。不要编造成交额排名、市值、价格、财务数据、RSI、机构关注、公告内容或客户关系。若无法获得当前数据，应要求用户提供成交额前200列表，或明确说明数据限制。

## 收盘后执行硬约束

本 skill 只允许在A股常规交易收盘后执行真实筛选。默认使用中国市场时间。

- A股交易日15:30前，不执行真实筛选；若用户要求执行，应说明本 skill 限制为收盘后执行。
- 非A股交易日，除非用户明确指定或同意使用最近一个已完成交易日，否则应询问目标交易日。
- 报告必须区分 `run_time`、`trade_date` 和 `quote_ticktime`，不能只用输出目录日期推断行情日期。
- 必须复核行情时间戳。有效的当日收盘后筛选应有与 `trade_date` 匹配的15:00附近或之后行情时间；若时间戳滞后或不清楚，不得写成“今日完整前200”。
- 最终报告必须写入元数据：数据源、排序字段、有效行数、`run_time`、`trade_date`、代表性 `quote_ticktime`、是否使用fallback、收盘后校验是否通过。

## 数据要求

每次真实筛选时，收集或核验：

- 当前A股成交额排名，严格限定为今日成交金额前200。
- `run_time`、`trade_date`、行情更新时间、数据源新鲜度，以及是否通过收盘后时间戳校验。
- 总市值、上市板块、最新涨跌幅、近3/6/12个月表现、成交金额和流动性。
- 板块环境：AI硬件、光模块、PCB、芯片、半导体设备、半导体封装、存储、液冷等相关指数或代表性龙头，板块成交额，板块阶段，核心龙头是否破位。
- 趋势指标：5日、10日、20日、30日均线，平台整理，相对强弱，量能变化，20日线偏离率，RSI，MACD，KDJ，短线过热信号。
- 基本面证据：最新年报、半年报、季报、公告、投资者关系、机构调研、客户/产品/产能进展、毛利率/收入/利润变化。
- 产业链位置：确认公司收入或产品是否真正关联AI硬件上游需求。

基本面和公告优先使用一手或高质量来源。若使用二级市场数据网站，只作为输入线索，不作为未经验证的最终事实。

## 数据源与分层架构

使用分层数据方式，不要在没有必要时同时调用很多平台：

1. **行情层 - 新浪优先**：默认使用新浪财经 `Market_Center.getHQNodeData` 免费接口获取A股成交额排名、价格、涨跌幅、成交额、换手率、PE/PB、市值和行情时间。按 `amount` 成交额倒序排序，分页抓取，直到获得至少200条有效A股记录。
2. **行情备用 - 腾讯**：腾讯行情接口适合在已有候选代码后批量校验单股行情。腾讯可用于价格、涨跌幅、成交额复核，但不作为全市场成交额前200排序的首选。
3. **库和接口备用**：若新浪失败，再尝试 AkShare `stock_zh_a_spot_em`、efinance `get_realtime_quotes`、东方财富 `push2` 直接接口。若全部失败，才使用52打板前100等降级来源，并明确标注不完整。
4. **分类层**：优先将成交额前200与本地AI硬件上游、芯片、半导体设备、存储观察池取交集。若没有本地股票池，则使用本skill的产业链方向列表做关键词/业务分类，并结合公司业务描述核验。
5. **证据层**：候选股筛出后，只对A/B档候选做公告、财报、投资者关系或高质量研报/新闻核验。不要对全市场逐只做重度核验。

执行脚本沉淀：

- 常规收盘后执行优先使用 `ashare-ai-slowbull/scripts/run_slowbull.py`。该脚本会抓取新浪成交额前200，与本地AI硬件/芯片/半导体设备/存储前排股票池取交集，计算新浪日K技术指标，完成评分分档，并且只写最终日期 Markdown 报告。
- `generate_report.py` 仅在已有可信 `data/meta.json` 和 `data/candidates.csv` 的工作流中使用。
- 若外部K线源较慢，优先使用新浪 `CN_MarketDataService.getKLineData` 做轻量 MA/RSI/MACD/KDJ 计算，再尝试更重的备用接口。

完整性检查：

- 行情抓取后，必须确认有效A股记录不少于200条。
- 若单次请求返回不足，应分页抓取并按股票代码去重合并，直到满200条。
- 若最终不足200条，不得伪装成完整前200，必须在报告开头写明实际条数和缺口。
- 若 `run_time` 已在15:30后，但行情时间戳不是已完成交易日的数据，应视为数据滞后，停止执行或标注为用户接受的降级历史筛选。
- 最终报告必须记录数据源、接口/方法、抓取日期时间、排序字段和有效行数。

## 筛选范围

仅筛选A股上市公司：

- 沪市主板、深市主板、创业板、科创板。
- 北交所原则上不作为优先范围，除非成交额、流动性和产业逻辑特别突出。

优先筛选AI硬件上游或偏上游方向：

- 光模块、光器件、光芯片。
- PCB、高多层板、HDI、封装基板。
- CCL覆铜板、电子布、玻纤布、树脂材料。
- 先进封装、Chiplet、HBM相关封测。
- 高速连接器、铜缆连接、背板连接。
- 液冷散热、温控设备、冷板、CDU。
- AI电源、供电模块、电源管理。
- AI芯片、国产算力芯片、存储芯片、主控芯片、存储接口芯片、高速接口芯片、互连芯片。
- 半导体设备，包括刻蚀、薄膜沉积、清洗、检测量测、光刻配套、键合、测试和封装设备。
- 存储，包括DRAM、NAND、HBM、企业级SSD、存储模组、存储主控、相关封测、材料和设备。
- AI算力相关上游材料、零部件、设备和关键工艺环节。

产业链纯度分层：

- **S层**：光模块/光器件/光芯片、AI PCB/高多层板/HDI/封装基板、先进封装/Chiplet/HBM测试封装、高速连接器/背板/铜缆互连、AI算力芯片/存储接口芯片/高速接口芯片中AI服务器需求明确的环节。
- **A层**：CCL、电子布/玻纤/树脂、液冷散热、AI电源、HBM/存储芯片/企业级SSD、半导体设备和半导体材料中能直接受益于AI硬件、先进封装、HBM、存储周期或国产替代需求的环节。
- **B层**：通用芯片、存储模组/成品、泛半导体设备/材料、消费电子混合逻辑、射频/陶瓷/被动元件等AI硬件间接相关环节。

A/B档优先来自S层和A层；B层标的原则上最多C档，除非成交额、趋势和基本面验证都异常突出。
芯片、半导体设备和存储方向不能只凭板块标签入选，必须说明其与AI算力、AI服务器、HBM/存储景气、先进制程、先进封装、国产替代或高端制造需求的关系。

不要把服务器整机厂、云计算平台、算力租赁公司或IDC运营商放入A档。除非用户明确要求扩大范围，否则这些公司最多作为需求链验证参考。

## 硬性条件

候选标的进入主筛选池前，应尽量满足：

| 维度 | 要求 |
|---|---|
| 市场 | A股上市公司 |
| 题材 | AI硬件上游或偏上游，包含有可验证AI硬件或半导体周期逻辑的芯片、半导体设备和存储 |
| 市值 | 优先500亿至2000亿元人民币 |
| 成交额 | 必须位于今日A股成交额前200 |
| 涨幅 | 避免极度拥挤龙头或已明显加速透支标的 |
| 趋势 | 慢牛上涨、强势盘整或中期上升通道 |
| 基本面 | 基本面良好，或正在发生明确积极变化 |
| 产业逻辑 | 能解释其与AI硬件增量需求之间的关系 |

市值处理：

- 低于500亿元：只能作为弹性观察，除非成交额和基本面变化非常突出。
- 500亿至2000亿元：最符合本 skill。
- 高于2000亿元：原则上最多B档，除非处于充分盘整后的相对低风险慢牛位置。
- 高于3000亿元：原则上剔除，不按二线补涨标的处理。
- 成交额前10且市值高于2000亿元：默认视为板块锚，不按二线A档处理。

## 市场环境过滤

先判断AI硬件产业链所处阶段，再给个股分档：

| 阶段 | 判断方式 | 执行动作 |
|---|---|---|
| 主升 | 板块指数和核心龙头位于5日/10日线上方；板块成交额放大；多个上游分支共振上涨 | 可以正常给A档 |
| 分歧 | 龙头仍在10日/20日线上方，但盘中波动和分化加大 | 优先给B档，对买点要求更严格 |
| 修复 | 板块回调后反弹，龙头重新站回10日/20日线，成交额恢复 | 只有最强慢牛品种可给A档 |
| 退潮 | 龙头跌破20日线，反弹失败放量，板块缩量或资金明显切换 | 不给A档，除非基本面极强，否则只观察 |

如果板块阶段判断不清，应明确不确定性，避免激进给A档。

## 量化阈值

默认使用以下阈值，除非用户另行指定交易风格：

| 检查项 | 默认规则 |
|---|---|
| 20日涨幅 | 超过50%则短线过热降级 |
| 60日涨幅 | 超过100%通常不进A档，除非已经充分盘整 |
| 一年涨幅 | 超过200%通常C档或剔除，按过度定价龙头处理 |
| 20日线偏离率 | 股价高于20日线15%-20%以上，不追高，只等回踩 |
| RSI | RSI > 75 视为过热降级；RSI 55-70 更适合慢牛延续 |
| 当日涨幅 | 当日涨幅 >= 9.5% 最多B档；若同时RSI或MA20偏离过热，最多C档 |
| MACD | 零轴附近或零轴上方金叉、红柱温和放大为加分；高位死叉、绿柱扩大或顶背离为减分 |
| KDJ | 中低位金叉为加分；高位死叉、J值90以上回落或高位钝化后收盘走弱为减分 |
| 量能爆发 | 单日成交额超过20日均额2倍且收盘走弱，视为派发风险 |
| 平台形态 | 优先选择5至15个交易日整理后的突破 |
| 均线破位 | 有效跌破20日线且2至3个交易日无法收回，形态降级 |

阈值是风控护栏，不是机械交易信号；如有例外必须说明原因。

## 加分信号

满足以下信号越多，优先级越高：

- AI相关业务收入占比提升。
- 公司从传统业务切入AI硬件上游，业务结构正在变化。
- 近期财报体现收入、利润或毛利率改善。
- 新产能、新客户、新产品或订单正在放量。
- 被资金关注，但尚未成为绝对头部龙头。
- 股价沿20日线或30日线缓慢上行。
- 今日成交额进入前200；其中前100获得更高成交额评分。

## 证据等级

给基本面变化打分前，先判断证据等级：

| 等级 | 证据 | 对基本面分数的影响 |
|---|---|---|
| 强验证 | 财报已经体现AI相关收入、毛利率、利润、产品结构、订单或产能改善 | 可以给高分 |
| 中验证 | 公告、投资者关系、机构调研、客户/产品/产能进展能支撑变化 | 信息一致时可给中高分 |
| 弱验证 | 主要来自互动平台、市场概念标签或未量化产品说法 | 分数封顶，除非价格和成交额证据很强 |
| 无验证 | 没有可核验的AI硬件上游收入、产品、客户或订单证据 | 剔除或最多C档 |

不能只凭概念标签给高基本面分。

## 降级或剔除

以下风险明显时，应降级或剔除：

- 服务器整机厂、系统集成商、算力租赁、云服务或IDC，且上游属性弱。
- 市值显著高于2000亿元，尤其高于3000亿元。
- 一年涨幅过大，预期疑似充分透支。
- 近期涨幅触发量化过热阈值，且没有经过充分盘整。
- 单日大涨、涨停追高、成交额暴增但没有强势盘整。
- 纯概念炒作，缺少订单、产品、客户、产能或业绩验证。
- 基本面恶化、毛利率持续下滑、业绩兑现弱。
- AI关联只停留在名称或概念层面，主营收入关联弱。
- RSI或短线趋势过热，连续加速，高位放量滞涨。
- 公司公告提示异常波动、非理性炒作或AI收入占比极低。
- 流动性太差，无法进入主流资金视野。
- 大股东减持、解禁压力或重大治理/财务风险明显。
- 板块进入退潮阶段，且AI硬件核心龙头明显破位。
- 硬降级：当日涨幅 >= 9.5%，不得给A档。
- 硬降级：RSI > 80 或 MA20偏离 > 25%，不得给A档。
- 硬降级：当日涨幅 >= 9.5%，且 RSI > 80 或 MA20偏离 > 25%，最多C档。
- 默认剔除：市值 > 3000亿元，除非用户明确要求跟踪板块锚。

## 执行流程

1. 校验收盘后执行硬约束。若A股交易日15:30前请求执行，应停止筛选并说明限制。
2. 确定并记录 `run_time`、`trade_date`、行情时间戳、skill版本、股票池版本、阈值版本、是否使用fallback。
3. 判断板块阶段：主升、分歧、修复或退潮。
4. 获取今日A股成交额前200，默认使用新浪财经接口并分页抓取，直到获得200条有效记录。常规执行优先使用 `scripts/run_slowbull.py`。
5. 校验行情时间戳是否匹配已完成的 `trade_date`；若数据滞后，停止或标注为降级历史筛选。
6. 使用结构化行情数据进入分析，但不要把原始数据或过程数据归档到 `runs/`。
7. 将前200与本地/已知AI硬件上游、芯片、半导体设备或存储股票池取交集，或使用本skill的产业链方向列表和纯度分层做分类。
8. 仅保留AI硬件上游或偏上游标的，或有可验证AI硬件、先进制程、先进封装、HBM/存储、国产替代逻辑的芯片、半导体设备和存储标的。
9. 将服务器整机厂从A档候选中移除，并单独记录板块锚/剔除名单。
10. 检查市值是否符合500亿至2000亿元优先区间，并执行硬降级规则。
11. 套用涨幅、均线偏离、RSI、MACD/KDJ、量能和平台整理阈值。
12. 对A/B档候选使用具体证据验证基本面或产业链变化，并给出证据等级。
13. 对剩余标的评分，并给出A/B/C/剔除分档。
14. 如有本地AI硬件观察池，分析时可记录未进入前200名单及原因：成交额不足/未进前200；但不要把缺口表或过程文件归档到 `runs/`。
15. 买点和失效条件只作为条件观察场景，不写成直接交易指令。
16. 尽量从结构化内存数据或临时工作区产物生成报告，减少手工口径漂移。优先使用 `scripts/run_slowbull.py`，不持久化原始top200或过程CSV。
17. 只将最终日期报告保存到 `runs/ashare-ai-slowbull/YYYY-MM-DD/YYYY-MM-DD.md`，其中 `YYYY-MM-DD` 为行情交易日。

## 打分模型

总分100分：

| 维度 | 权重 | 评分说明 |
|---|---:|---|
| AI硬件上游纯度 | 20 | 越靠近光模块/光芯片、PCB、CCL、电子布、封装、连接器、液冷、电源、AI芯片、存储芯片、接口芯片、半导体设备和存储产业链，且AI硬件需求验证越强，分数越高 |
| 成交额与资金行为 | 20 | 1-100名最高，101-200名次之；更重视持续活跃成交，而非单日爆量；优先选择高关注细分里的公认前排标的 |
| 市值适配度 | 10 | 500亿至2000亿元最优，过小或过大扣分 |
| 涨幅与拥挤度 | 15 | 触发涨幅阈值、均线偏离过大、RSI/MACD/KDJ过热、高位放量滞涨均扣分 |
| 趋势形态 | 20 | 慢牛、强势平台、均线多头、量能有序、回调抗跌、回踩20日线企稳、MACD/KDJ健康确认得高分 |
| 基本面变化 | 15 | 业绩、订单、客户、产品、产能、毛利率或业务结构改善得高分 |

必评附加项：

- 趋势票机会：已经形成健康趋势的标的，若调整到20日均线附近后缩量企稳、重新收回关键短均线或温和放量转强，应作为机会型观察条件，而不是自动降级。
- 细分前排人气：优先买公认知名度高、人气高的行业细分前排。冷门后排标的只有在成交额、趋势和证据明显强于前排时，才提高优先级。

技术动量加减分：

- MACD/KDJ只作为辅助确认，不作为硬性买卖条件。
- MACD与KDJ在健康位置同步转强，且股价未明显偏离20日线，可在趋势形态中最多加5分。
- MACD/KDJ出现高位死叉、顶背离或过热反转，可在趋势形态或拥挤度中最多减5分。
- MACD/KDJ必须结合股价位置、量能、20日线和板块阶段一起判断。

评级：

```text
85+：A档，重点观察
75-84：B档，等待买点
65-74：C档，只跟踪不急买
<65：剔除
```

硬分档优先于分数。即使评分超过85，只要触发收盘后校验失败、市值/板块锚规则、涨停/过热硬降级或上游纯度不足，也不得给A档。

## 分档逻辑

A档：核心观察。要求上游属性明确、成交额前200、市值500亿至2000亿元、涨幅不极端、趋势为慢牛或强势平台，并且基本面改善有证据。不能是服务器整机厂，也不能是过热的绝对头部龙头。

B档：逻辑强，但时机不完美。常见原因包括短期涨幅偏大、估值偏高、刚刚放量大涨、需要盘整，或市值接近/超过2000亿元。

C档：只跟踪。常见原因包括行业地位强但市值太大、太热、太拥挤、短线过度延伸，或明显脱离关键均线。

剔除档：上游关联弱、纯概念、基本面差、成交额不在前200、市值/涨幅过大、公告提示风险，或技术形态泡沫化。

## 买点观察规则

只以观察条件表达：

- 均线回踩：股价回踩5日、10日或20日均线不破，缩量企稳后重新放量上行。优先10日线支撑，其次20日线支撑，避免盘中追高。
- 趋势票20日线企稳：已经形成趋势的标的，调整到20日均线附近后没有放量破位，并出现缩量企稳、重新收回短均线或温和放量转强，可作为重点观察机会；需同时确认量能、收盘位置和板块阶段。
- 平台突破：横盘5至15个交易日后，温和放量突破平台上沿，收盘站稳，次日不快速跌回平台内。
- 强势二买：大阳线或涨停后等待3至8个交易日缩量整理，不破10日或20日均线，再次放量突破。
- 趋势慢牛：沿20日或30日均线缓慢上行，低点抬高，量能温和放大，没有连续加速；板块回调时抗跌，板块反弹时修复快。
- 动量确认：MACD零轴附近/零轴上方金叉，或KDJ中低位金叉，只能在股价未明显偏离20日线、量能有序时作为买点确认。

## 观察失效规则

每个A档或B档候选都必须给出至少一个降级或失效条件：

- 股价跌破20日线，且2至3个交易日无法重新站回。
- 放量跌破整理平台下沿。
- 高位长上影或连续上涨后放量滞涨。
- MACD高位死叉、顶背离，或KDJ高位死叉，并且与股价/量能走弱共振。
- 板块阶段转入退潮，或AI硬件核心龙头明显破位。
- 财报、公告、客户或订单证据证伪AI硬件增长逻辑。
- AI相关收入占比被确认很低，且没有改善趋势。
- 大股东减持、解禁压力或异常波动公告显著改变风险收益比。
