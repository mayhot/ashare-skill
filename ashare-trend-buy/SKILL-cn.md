# A股右侧趋势买入标准

本技能用于把 A 股候选标的整理成纪律化研究观察池。目标不是预测最低点，而是寻找已经进入上升趋势、回调或突破结构健康、处在当前市场主线中，并且基本面或产业逻辑能解释的标的。

输出必须使用“观察、等待确认、若跌破则剔除、条件满足后再评估”等条件化表达。不得写成直接买入建议。不得编造成交额、均线、指标、财报、公告、行业催化或资金流数据。

## 回测后的胜率修正规则

2026-05-20 至 2026-05-27 的推荐标的回测显示：B 档的至今收益和 5 日收益优于 A 档，C 档至今均值接近 0，且 5月25日、5月26日同日涨幅较高的标的后续明显回落。因此后续筛选以提高胜率为优先，不以凑满 A 档数量为目标。

- A 档必须更稀缺：只保留当前验证主线前排、不过热、靠近支撑、量能可控的标的。
- B 档作为主要等待池：趋势和逻辑较好但尚未出现规则内买点的标的，宁可放 B，不提前抬 A。
- C 档只跟踪：不得表达成正式短名单，不作为买点观察池。
- 过热跟踪单列：单日涨幅、20日线偏离、换手、量比或 KDJ 过热时，不追高，只等待 3-8 个交易日缩量整理。
- 冷却观察：最近回测中同一标的多次推荐后持续走弱，至少降到 C 档，不能因单日修复重新进入 A/B。
- 主线纯度提高：半导体、CPO/光模块、PCB、AI服务器、存储、高速连接器、液冷等验证主线优先；资源、材料、机器人/汽车零部件、量子等分支提高门槛。

## 数据纪律

必须先做数据获取和指标计算，再写报告。

1. **候选池和流动性**
   - 优先使用腾讯或新浪的公开行情/排行接口。
   - 若腾讯/新浪无法提供所需成交额或候选池，再用东方财富作为 fallback。
   - 也可加入用户提供股票池、当前主线龙头、近期突破标的、缩量回踩企稳标的。
2. **日K/OHLC**
   - 优先使用腾讯日K数据计算均线、MACD、KDJ。
   - 腾讯不可用时，优先尝试新浪日K数据。
   - 再使用 BaoStock 或 AKShare 作为备用。
   - 东方财富历史K线只能作为最后备选，因为接口可能不稳定。
3. **主线和新闻**
   - 收评、政策文件、交易所公告、财报、行业数据用于判断市场环境和主线。
   - 新闻不能用于编造技术指标。
4. **完整性门槛**
   - 日K最后日期应等于筛选日期；若滞后，必须说明。
   - 每只股票至少需要 60 根日K，才能做完整技术筛选。
   - 如果没有连续 OHLC，结果只能标为“初筛观察池”；MACD/KDJ 必须写“未取得K线，未计算”；这类标的不能进入正式 A 档。

## 指标计算脚本

优先使用同目录脚本。脚本默认 `--source auto`，会先尝试腾讯日K，失败后再尝试 BaoStock：

```powershell
python ashare-trend-buy/scripts/calc_indicators.py --codes 000988,300274,688012 --end-date 2026-05-20 --source auto
```

若缺依赖，可在项目内安装：

```powershell
python -m pip install --target .deps baostock pandas
```

脚本输出 JSON，包含收盘价、MACD 标签、DIF/DEA/HIST、KDJ 标签、K/D/J。

## 指标公式

默认使用：

- MACD：EMA12、EMA26、DEA9；`DIF = EMA12 - EMA26`；`HIST = 2 * (DIF - DEA)`。
- KDJ：9日 RSV；K、D 按 `2/3 * 前值 + 1/3 * 当前值` 平滑；`J = 3K - 2D`。

输出指标时同时写状态和值，例如：`MACD多头延续；KDJ多头，K=82.37/D=79.44/J=88.23`。

MACD/KDJ 只做辅助确认或风险提示，不能单独触发买点。

## 核心规则

1. **先匹配交易周期**：短线看 5/10/20 日均线；波段或中线补看 20/30/60 日结构。
2. **只在趋势向上时观察**：股价应站上关键均线，均线多头排列或正在向多头修复。
3. **回调企稳再观察**：大涨后优先等待缩量回调、止跌、小K线震荡。
4. **右侧确认**：突破前高/平台并收稳，或回踩支撑有效后温和放量转强，才算可观察结构。
5. **量价健康**：上涨温和放量，回调明显缩量；剔除高位巨量上影、放量大跌、反复假突破、量价背离。
6. **主线优先但不追拥挤**：优先政策、业绩、产业趋势或题材主线；避开连续加速、远离20日线、巨量滞涨、情绪高潮。
7. **客观定义龙头**：看板块相对强度、成交额、率先突破、回调抗跌、修复速度、产业逻辑纯度，不只看单日涨幅。
8. **趋势票看20日线企稳机会**：趋势票大涨后不追高，优先观察调整幅度是否回到或接近20日均线；缩量止跌、收盘不破或跌破后1-2日快速收回，才视为可评估机会。
9. **优先行业细分前排**：同等技术结构下，优先选择公认知名度高、人气高、成交活跃、产业逻辑清晰的行业细分前排，回避弱跟风和辨识度不足的后排。
10. **剔除必须解释原因**：高分但被剔除的标的，必须写清原始评分、硬性剔除原因和风险调整后档位，避免读者误以为是排序错误。
11. **必须有支撑和失效条件**：支撑可为关键均线、前低、箱体下沿、平台突破位。失效优先用收盘跌破和1-2日修复情况判断。
12. **仓位逻辑分批**：首次只能作为轻仓试错观察，走对后再评估加仓；不暗示重仓或固定比例。
13. **逻辑必须能解释**：证据优先级为财报 > 公告/订单/产能/产品 > 政策/行业数据 > 机构调研 > 媒体传闻。
14. **看大盘和板块环境**：大盘或板块破位时提高门槛，只保留最强 A 档观察。

## 硬性剔除

出现以下情况应剔除或降级：

- 股价在 20 日线下方，5/10/20 日均线仍空头。
- 刚刚暴涨，尚未缩量回调或平台整理。
- 高位巨量长上影、放量大阴线、突破失败。
- 连续加速、远离20日线、巨量滞涨，或处在明显情绪高潮。
- 回调不缩量，反弹一次比一次弱。
- 不在当前主线中，只是弱跟风。
- 没有清晰支撑和失效条件。
- 纯概念炒作，没有财报、公告、订单、政策、产品、产能或行业证据。
- 有异常波动、减持压力、治理/财务风险、监管风险或重大利空。

## MACD/KDJ 降级规则

- `MACD多头延续 + KDJ多头且J未过热`：可辅助 A/B 档，但仍需趋势和量价配合。
- `MACD多头但动能收敛`：不能当买点，只能等待缩量整理或再突破。
- `MACD空头/动能走弱`：不得保留 A 档，最多 B/C，除非价格结构明显快速修复。
- `MACD死叉`：至少降一级；若在零轴下方死叉或跌破支撑，剔除或仅 C 档。
- `KDJ高位死叉`：短线至少降一级。
- `KDJ高位偏热` 或 J 值持续大于 100：不追，等待整理。
- `KDJ金叉` 不能单独作为买点，必须配合趋势、支撑和量价。

## 评分模型

总分 100：

| 维度 | 权重 | 要点 |
|---|---:|---|
| 趋势结构 | 17 | 5/10/20日均线、多头排列、低点抬高、股价站上关键均线 |
| 回调/突破质量 | 13 | 缩量企稳、回踩或接近20日均线企稳、平台突破、突破后不快速跌回 |
| 量价健康度 | 13 | 上涨温和放量、回调缩量、无出货形态 |
| 主线强度 | 15 | 政策、业绩、产业趋势或题材主线，相对强度可验证；优先公认知名度高、人气高的细分前排 |
| 支撑与风控 | 15 | 支撑清晰，失效位明确，离支撑不太远 |
| 基本面/逻辑 | 15 | 财报、公告、订单、产品、产能、行业拐点或政策逻辑可验证 |
| 技术辅助指标 | 7 | MACD/KDJ辅助确认或风险提示 |
| 流动性 | 5 | 成交活跃，不太冷门 |

分档：

```text
A档：主线纯度高、不过热、靠近支撑、量能可控，等待规则内买点确认
B档：逻辑较好，但还需等待回调、缩量或突破确认
C档：只跟踪，不进入正式短名单
过热跟踪：强势但短线涨幅、20日线偏离、换手、量比或KDJ过热，不追高
剔除：硬性风险、评分不足、破位或低胜率结构
```

A 档额外门槛：单日涨幅不超过约 4%，距20日线不超过约 12%，换手不超过约 15%，量比不超过约 2.2，且未出现在最近回测连续走弱冷却名单中。

## 买点观察模型

买点只能写成条件观察：

- **均线回踩型**：回踩10日或20日均线，缩量不破，随后温和放量转强。
- **20日线企稳型**：趋势票调整到或接近20日均线，缩量止跌、收盘不破或快速收回20日线，再观察温和放量转强。
  - 强企稳：距20日线约 -1% 到 +5%，量能温和，MACD未明显走坏。
  - 可观察：距20日线约 -2% 到 +10%，量能未明显失控，等待止跌确认。
  - 不追：距20日线超过约18%，优先等待3-8个交易日整理靠近均线。
- **平台突破型**：横盘5-15个交易日后，放量突破平台高点或前高，收盘站稳，次日不快速跌回。
- **强势二买型**：大涨或涨停后等待3-8个交易日缩量整理，不跌破10/20日线，再次放量突破。
- **慢牛延续型**：沿20/30日均线缓慢上行，低点逐步抬高，板块回调时抗跌，板块反弹时率先修复。

每只标的必须写清支撑/失效位。优先使用收盘价或连续修复情况判断失效，避免被盘中瞬时击穿误导。

## 工作流程

1. 明确筛选日期、交易周期、市场状态和候选池来源。
2. 获取成交额/流动性和主线板块信息。
3. 获取每只候选的日K，先计算 5/10/20/30/60 日均线、量能均值、MACD、KDJ。
4. 若日K不完整，结果标为“初筛观察池”，不得输出正式 A 档。
5. 判断大盘和板块是否支持右侧交易，若破位则提高门槛。
6. 按硬性剔除条件先过滤。
7. 检查趋势、回调/突破质量、量价、支撑和拥挤度。
8. 应用 MACD/KDJ 加减分和降级规则。
9. 判断个股在板块中是龙头、二线强势、强跟随还是弱跟风。
10. 验证基本面或产业逻辑证据。
11. 打分并分为 A/B/C/剔除。
12. 输出候选表、逐个点评、最终短名单、风险提示。
13. 将本次运行的最终报告保存到 `runs/ashare-trend-buy/YYYY-MM-DD/YYYY-MM-DD.md`；过程数据只临时使用，不归档到 `runs/`。
14. 最后一步必须回测向前 10 天已有推荐结果，生成回测报告和明细 CSV，保存到 `runs/ashare-trend-buy/backtests/`；若历史报告不足 10 天，按实际可用报告回测并说明样本数。

## 运行结果保存

每次执行后必须新建日期目录：

```text
runs/ashare-trend-buy/YYYY-MM-DD/
```

目录中至少包含：

```text
runs/ashare-trend-buy/YYYY-MM-DD/YYYY-MM-DD.md
```

`YYYY-MM-DD.md` 保存最终展示给用户的报告。`runs/` 只保留每天的计算结果，不保留 `data/`、`scripts/`、原始接口响应、标准化候选池、指标 JSON 或评分中间表。

每次执行还必须生成最近 10 天推荐回测：

```text
runs/ashare-trend-buy/backtests/YYYY-MM-DD_trend_buy_START_to_END_backtest_report.md
runs/ashare-trend-buy/backtests/YYYY-MM-DD_trend_buy_START_to_END_backtest_report.csv
```

回测口径必须包含推荐当日、5 个交易日、至今涨跌幅。该回测报告属于结果报告，可以保存在 `runs/ashare-trend-buy/backtests/`；原始行情和中间 K 线仍不得归档。

若同日目录已存在，只有在新结果用于替换旧结果时才覆盖；否则询问用户是否覆盖，或另存为 `YYYY-MM-DD/HHmmss/` 时间戳子目录。
